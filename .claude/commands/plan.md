# Create Implementation Plan

Read the current state of the project:
1. Read `README.md` for project overview and goals
2. Read `docs/DEVLOG.md` for recent context and decisions
3. Check open GitHub issues with `gh issue list`

Then, for the topic: $ARGUMENTS

1. Break it into discrete tasks
2. Identify dependencies between tasks
3. For each task, note: what files are affected, what tests are needed
4. Create GitHub issues for each task using `gh issue create`
5. Label issues appropriately (enhancement, bug, feature, documentation, etc.)
6. Log this planning session in `docs/DEVLOG.md`
